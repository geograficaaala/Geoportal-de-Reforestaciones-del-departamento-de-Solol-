Módulo de Incendios forestales

Este módulo integra detecciones de fuego activo/anomalías térmicas (NASA FIRMS — Near Real-Time) y las organiza como información territorial consultable para apoyar monitoreo y coordinación. Incluye recorte por unidades de gestión (municipios, microcuencas y áreas protegidas), KPIs y lectura geoestadística.

Propósito del módulo
  • Explorar focos de calor por periodo (24h / 72h / 7d / 30–90d) y sensor (VIIRS/MODIS NRT).
  • Priorizar atención territorial por unidad administrativa o ambiental.
  • Interpretar patrones espaciales (clústeres, escala, dispersión) para apoyo operativo.
  • Documentar insumos para coordinación interinstitucional (sin almacenar histórico en el repositorio).

Nota: FIRMS reporta “fuego activo” (hotspots) y FRP; no es perímetro ni área quemada. Requiere validación operativa local.

Metodología de trabajo

Cómo se consulta FIRMS sin exponer la MAP_KEY y se entrega al geoportal

Flujo técnico

1
Consulta NRT
El geoportal solicita al proxy (Cloudflare Worker) datos recientes por periodo y sensor.
2
Proxy seguro
La MAP_KEY se guarda como Secret en Cloudflare; se aplica cache corto y CORS para GitHub Pages.
3
Normalización
Se unen ventanas temporales, se deduplica y se entrega JSON uniforme con resumen y filas.
Limitaciones técnicas
  • La detección depende de nubosidad/humo y del paso orbital (no es monitoreo continuo).
  • VIIRS (~375 m) y MODIS (~1 km) son resoluciones de píxel: ubicación aproximada.
  • FRP (MW) indica potencia radiativa; no equivale a área quemada.

Recorte por unidades territoriales

El geoportal interpreta los puntos en relación con capas de referencia. Esto permite lecturas comparables y reportes por unidad de gestión.

  • Municipios (Sololá)
  • Microcuencas (cuenca Atitlán)
  • Áreas protegidas (CONAP y polígonos de referencia)
Las capas se publican como GeoJSON y se usan para recorte espacial y estadísticas por zona.

Análisis territorial para apoyo a monitoreo

De detecciones a lectura espacial operativa

El módulo permite ubicar concentraciones de focos de calor por periodo y evaluar consistencia por sensor. La lectura territorial prioriza zonas recurrentes y facilita coordinación entre instituciones para verificación.

1
Filtrado por periodo y sensor
Comparación VIIRS/MODIS NRT, series diarias y FRP para evaluar intensidad relativa.
2
Cruce por unidad territorial
Conteos por municipio/microcuenca/área protegida para reportes rápidos y priorización.
3
Salida para coordinación
Tabla exportable (Excel/CSV) y GeoJSON para integrarse a otros flujos SIG.
Uso recomendado
  • Verificar con reportes locales (CONRED/UGAM/brigadas) y condiciones meteorológicas.
  • Usar “Alta confianza” para priorización rápida y revisar “Nominal” para contexto.
  • Interpretar clusters como señal de persistencia/repetición, no como perímetro.

Preguntas clave

  • ¿Qué unidades territoriales concentran más detecciones recientes?
  • ¿Hay recurrencia por varios días (persistencia) o eventos puntuales?
  • ¿Qué sensores confirman el patrón (VIIRS vs MODIS)?
  • ¿Cuál es la escala espacial dominante del agrupamiento?

Resultado esperado

Lectura espacial rápida para apoyo a coordinación: “dónde, cuánto, cuándo y con qué confianza”, con evidencia estadística básica.

Geoestadística aplicada

Patrones espaciales (CSR/NNI/Ripley/variograma) para interpretar distribución

La geoestadística ayuda a responder si las detecciones se comportan como un patrón aleatorio (CSR) o si existe agrupamiento (clusters) significativo a ciertas escalas. También resume dispersión (SDE) y continuidad de FRP (variograma).

Lecturas típicas
  • NNI y z/p: evidencia de agrupamiento vs CSR a escala de vecino más cercano.
  • Ripley L(r): evaluación multiescala; identifica una distancia r* con mayor señal.
  • SDE: dispersión y orientación (anisotropía) de la nube de detecciones.
  • Variograma de FRP: indicios de autocorrelación espacial de intensidad radiativa.
Estos resultados apoyan interpretación, pero no sustituyen verificación en campo ni análisis de combustibles/cobertura.

Buenas prácticas

1
Control de periodo
Comparar 24–72h vs 7–30d para separar eventos puntuales de recurrencia.
2
Control de sensor
Cruzar VIIRS y MODIS para reducir falsos positivos y entender diferencias de resolución.
3
Control territorial
Interpretar por municipio/microcuenca/área protegida para asignación de responsabilidad y coordinación.
4
Validación
Integrar reportes locales, meteorología y condiciones de combustible/cobertura forestal.
